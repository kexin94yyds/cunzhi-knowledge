# Agent 架构演进与进步指标

> 来源：Gemini 对话记录
> 日期：2026-01-17

## 四大衡量维度

### 1. 更长的线程 (Longer Threads)
- **定义**：AI 注意力跨度更长，能在长任务中不丢失最初的理解
- **衡量**：Agent 能连续执行多少个有效步骤而不因上下文遗忘或幻觉中断
- **进步体现**：从「每一步都检查」变成「长任务链结束后再检查」

### 2. 更厚的线程 (Thicker Threads)
- **定义**：AI 能获取多维度信息（代码库、API 文档、数据库 Schema）
- **衡量**：AI 能同时调用多少种工具/数据源来完成任务
- **进步体现**：从「单薄的文本理解」变成「立体的多维理解」

### 3. 更多的线程 (More Threads)
- **定义**：并发执行多个任务，管理多个独立工作流
- **衡量**：用户能同时管理多少个并行任务
- **进步体现**：从「串行处理」变成「并发管理」

### 4. 更少的人工介入点 (Fewer Human Interventions)
- **定义**：减少用户需要干预的次数
- **衡量**：完成任务需要多少次人工确认/修正
- **进步体现**：从「每步确认」变成「自主闭环」

## 与 iterate 的对齐

| 维度 | iterate 实现 |
|------|-------------|
| 更长的线程 | context.md + patterns.md 保持上下文 |
| 更厚的线程 | MCP 工具 + codex exec 多源信息 |
| 更多的线程 | multi-agent-dispatch 并发执行 |
| 更少的介入 | zhi 循环 + 自动沉淀三件套 |
