# 问题事实库 (Problems Registry)

> 统一记录所有项目中发生过的问题，作为跨项目长期经验积累。

---

## 条目格式说明

每个问题条目应包含：
- **ID**: P-YYYY-NNN 格式
- **项目**: 项目名称
- **仓库**: git 地址
- **发生版本**: commit hash 或 tag
- **现象**: 用户可见的表现
- **根因**: 技术层面的原因
- **修复**: 采用的解决方案
- **回归检查**: 关联的 R-ID
- **状态**: 已修复 / 待修复 / 观察中
- **日期**: YYYY-MM-DD
