# docs-list Skill

先读 docs 索引或文档清单，再开始任何改动。适合中大型项目或上下文不清晰时使用。

## 核心原则

1. 先用文档索引或脚本获取结构，再做改动。
2. 只读取与任务相关的 docs，避免全量加载。
3. 输出必须引用具体文档路径。

## 推荐流程

1. 运行 docs 索引脚本（如存在）：
   - `docs:list`
   - `./scripts/docs-list`（或项目内约定）
2. 选择 2-5 个最相关的 docs 文件
3. 摘要要点并确认理解
4. 再进入实现或改动

## 触发时的默认行为

- 如果没有 docs 索引脚本，先搜索 `docs/` 目录结构并列出候选文档。
- 对每个候选文档给出“可能相关”的理由。

## 示例提示

"先运行 docs:list 并读相关文档，再动手改。"
