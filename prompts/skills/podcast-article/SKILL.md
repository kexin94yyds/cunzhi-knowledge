# 播客和公众号文章工作流

将语音录入内容转化为播客稿和公众号文章的完整工作流。

**核心原则：分步进行，每一步都要用户确认。**

---

## 工作流概览

```
语音转文字 → 结构化(确认) → Markdown(确认) → 播客稿(确认) → 公众号文章
```

---

## 第一步：原稿 → 结构化

用户提供语音转文字的原稿后，AI 先输出**结构化大纲**：

- 提取核心观点
- 分章节整理
- 列出要点

**等待用户确认结构是否 OK。**

---

## 第二步：结构化 → Markdown 格式

结构确认后，AI 输出**完整 Markdown 文稿**：

- 按确认的结构展开
- 保留原稿的核心内容和语气
- 格式清晰，便于阅读

**等待用户确认 Markdown 是否 OK。**

---

## 第三步：Markdown → 播客稿

Markdown 确认后，AI 将其**打磨成播客稿**：

- 口语化，适合讲述
- 有节奏感，短句为主
- 有互动感，可加问题收尾

**等待用户确认播客稿是否 OK。**

---

## 第四步：播客稿 → 公众号文章

播客稿确认后，AI 将其**转化为公众号文章**：

- 适合阅读的文章格式
- 段落分明，金句突出
- 可加小标题、分割线

---

## 关键提示词

| 步骤 | 用户提示词 |
|------|-----------|
| 结构化 | `请把以上的文字结构化一下，我要做成播客讲给大家。` |
| 扩展内容 | `全面且结构化，太短了啊` |
| 继续生成 | `继续` |

---

## 注意事项

1. **不要一次性生成**：每一步都要等用户确认
2. **保持原稿风格**：不要过度改写用户的表达
3. **结构先行**：先确认结构，再写内容
4. **分步打磨**：播客稿和公众号文章是两种不同的文体
