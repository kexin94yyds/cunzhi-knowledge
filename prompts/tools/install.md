# /install

**模板分类**: Action

用于配置新环境或验证依赖健康。无需参数。

## 步骤

1. **同步仓库**: 
   - `git fetch --all --prune`
   - 切换到 `develop`
   - `git pull --rebase`
   - 确认干净工作区

2. **了解项目**: 
   - 运行 `/prime` 理解仓库布局
   - 了解关键文档和工具期望

3. **验证运行时**: 
   - 捕获 `node --version`
   - `bun --version`
   - 确保支持的工具链可用

4. **安装依赖**: 
   - 执行 `bun install`
   - 捕获两次运行的日志

5. **管理环境文件**: 
   - 复制 `.env.sample` 到 `.env`（如缺失）
   - 填写必需密钥
   - 确认 git 保持干净

6. **可选工具**: 
   - 注意补充设置（如 Docker）
   - 验证可用性

7. **最终验证**: 
   - 重新运行 `git status --short`
   - 列出关键命令
   - 记录任何待完成设置任务

## 报告

- 记录的工具版本
- 安装命令结果和日志路径
- 环境文件状态和剩余行动项
