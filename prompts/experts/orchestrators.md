# 专家编排器

多专家协调进行全面分析。

## planning_council - 计划委员会

协调所有专家进行需求分析。

### 输入
- `$ARGUMENTS`: 任务描述或需求

### 执行流程
1. 并行调用所有 4 个专家的 `_plan` 命令
2. 收集每个专家的分析结果
3. 综合发现，识别共识和分歧
4. 产出统一的计划贡献

### 输出格式
```
## 计划委员会分析

### 架构视角
[架构专家的分析摘要]

### 测试视角
[测试专家的分析摘要]

### 安全视角
[安全专家的分析摘要]

### 集成视角
[集成专家的分析摘要]

### 综合建议
[统一的优先建议列表]

### 跨切关注点
[所有专家共同识别的问题]

### 分歧点
[专家间的不同意见及解决建议]
```

## review_panel - 审查小组

协调所有专家进行代码审查。

### 输入
- `$ARGUMENTS`: PR 编号或 diff 上下文

### 执行流程
1. 并行调用所有 4 个专家的 `_review` 命令
2. 收集每个专家的审查结果
3. 聚合状态（任一专家标记 CHANGES_REQUESTED 则整体为 CHANGES_REQUESTED）
4. 产出单一整合审查决定

### 输出格式
```
## 审查小组决定

### 整体状态: APPROVE | CHANGES_REQUESTED | COMMENT

### 架构审查
[状态]: [关键发现]

### 测试审查
[状态]: [关键发现]

### 安全审查
[状态]: [关键发现]

### 集成审查
[状态]: [关键发现]

### 阻塞问题
[必须解决的问题列表]

### 改进建议
[可选但推荐的改进]

### 综合评价
[整体审查总结]
```

## improve_orchestrators - 编排器自我改进

分析编排器历史表现并改进。

### 执行流程
1. 回顾最近的编排器调用记录
2. 分析专家协调效率
3. 识别改进机会
4. 更新编排策略

### 输出
- 编排效率报告
- 策略改进建议
- 更新后的编排规则
