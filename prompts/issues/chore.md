# /chore

**模板分类**: Action

为 `$ARGUMENTS` 中描述的维护任务起草规范。杂务涵盖依赖更新、CI 调整、文档刷新或清理重构。

## 指令

- **首先验证问题标签**: 运行 `gh issue view <issue-number> --json labels`
- 在 `docs/specs/` 下创建规范，命名为 `chore-<issue-number>-<slug>.md`
- 从问题标题使用 3-6 个小写、连字符分隔的单词构建 `<slug>`
- 在规范顶部引用问题元数据用于追溯
- 列出所有受影响的文件和范围边界
- 遵循仓库 git 流程

## 规范格式

```md
# 杂务规范: <简洁名称>

## 目标
- 为什么需要此杂务
- 成功标准

## 范围
- 包含内容
- 排除内容

## 相关文件
- <path> — <为什么触及>
### 新文件
- <path> — <目的>

## 任务分解
### 准备
- <前置步骤>
### 执行
- <核心任务>
### 验证
- <确认步骤>

## 风险与缓解
- <风险> → <缓解>

## 验证命令
- lint
- typecheck
- test（如适用）
- build（如适用）
```

## 报告

**仅**返回规范文件路径作为纯文本单行。
